# v4.11 — Remaining follow-ups (post-ship)

**Context:** `v4.11.0` shipped 2026-06-09 (radial navigation + de-slop credibility pass — merged `#17`, tagged, released). These are the items deliberately deferred or surfaced during that work, captured so they aren't lost. None block the release; they're the next polish/feature increments.

---

## 1. ZATCA compliance module (deferred — needs data pipeline first) — **P2, blocked**

The e-invoicing pipeline does **not** exist yet, so this was intentionally not built (building it now = empty UI over absent data, the slop v4.11 removed).

- **Root gap:** `apps/web/app/actions/billing.ts` `generateSubscriptionInvoice` (~L315–345) creates invoices but never sets any ZATCA field → every `Invoice.zatcaStatus` defaults to `NOT_APPLICABLE`; `zatcaQrCode` / `xmlContent` / `zatcaSubmittedAt` / `zatcaClearedAt` stay null.
- **Schema is ready:** `packages/db/prisma/schema.prisma` — `enum ZatcaStatus { NOT_APPLICABLE, PENDING, CLEARED, REPORTED, REJECTED }` (~L997) and the `Invoice` ZATCA block (~L1182–1188). `REPORTED` (simplified-invoice reporting) is defined but unused.
- **Build order when the clearance integration lands:** (1) a clearance/reporting service that populates the fields on invoice issue; then (2) **tenant invoices page** (`apps/web/app/dashboard/billing/invoices/page.tsx`) — per-invoice clearance badge + QR + failed-submission retry (data already returned by `getInvoices`/`getInvoiceById`, just unrendered); then (3) **finance dashboard** (`finance/page.tsx`) aggregate compliance widget (clearance %, pending/rejected, cert health) — reuse `app/actions/admin-analytics/getZatcaClearanceRate.ts` which already returns `{rate, cleared, rejected, pending, last7Rejections, alertSpike}`.
- The existing platform-admin ZATCA-clearance KPI (`dashboard/admin/page.tsx:~774`) stays as the platform-health view.
- Also update `apps/web/lib/help-content.ts:~106` FAQ ("E-invoicing under development") once shipped.
