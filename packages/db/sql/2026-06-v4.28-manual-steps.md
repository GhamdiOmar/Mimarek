# v4.28.0 — Manual steps against the POPULATED database

> Schema-correctness change-set (QA-DB-01..06 + QA-SEC-02). These are the steps a human
> runs **in order** against every long-lived environment that already holds rows
> (production + any long-lived staging). CI's Postgres is ephemeral (0 rows) and needs
> none of this — production silently drifts from CI until each step below is applied.
>
> What this release changes structurally:
> - **Recast** every SAR-money `Decimal` to `@db.Decimal(14,2)` and the two VAT-rate
>   columns to `@db.Decimal(6,4)`. On Postgres this is an `ALTER COLUMN ... TYPE numeric`.
> - **Additive nullable columns:** `organizationId` on `Lease`, `Reservation`,
>   `PaymentPlanInstallment` (+ their indexes).
> - **New composite indexes** (`Customer`, `MaintenanceRequest`, `Reservation`,
>   `PaymentPlanInstallment` — tenant-leading).
> - **New unique constraint:** `CouponRedemption (couponId, organizationId)`.
> - **Cascade → Restrict** on `SubscriptionEvent.subscription`,
>   `SubscriptionMrrSnapshot.subscription`, `SubscriptionMrrSnapshot.organization`
>   (FK ON DELETE rule change; no data change).
>
> Mixed-case Prisma table names MUST be double-quoted in raw SQL (`"Lease"`, not
> `Lease`) — unquoted identifiers fold to lowercase and `ALTER TABLE IF EXISTS` then
> silently no-ops (AGENTS.md §4).

## Step 1 — Money-precision precheck (run BEFORE `db push`)

Casting `numeric` (unconstrained) → `numeric(14,2)` **truncates the scale to 2 decimals
and will ERROR if any value has more than 12 integer digits** (14 total − 2 fractional).
Existing app code already stores ≤ 2 decimals, but verify before the irreversible cast.

> **✅ PRECHECK ALREADY RUN against the local Supabase DB (2026-06-16, `scripts/precheck-v4280.mjs`).**
> Result: every money column is clean **except 5 `mrrSar` rows** — `Subscription.mrrSar` (4)
> + `SubscriptionMrrSnapshot.mrrSar` (1), all `399.16666…` (= a 4790-SAR annual plan ÷ 12,
> a legitimate computed MRR, NOT a data bug). No overflow anywhere; no `>2`-decimal value
> in any other column; **`CouponRedemption` has 0 rows → the new `@@unique` is safe**;
> backfill scope = Lease 2 / Reservation 7 / PaymentPlanInstallment 14 rows.
> **Remediation (run this BEFORE Step 3 so the cast is provably lossless):**
>
> ```sql
> BEGIN;
> UPDATE "Subscription"            SET "mrrSar" = round("mrrSar", 2) WHERE "mrrSar" <> round("mrrSar", 2);
> UPDATE "SubscriptionMrrSnapshot" SET "mrrSar" = round("mrrSar", 2) WHERE "mrrSar" <> round("mrrSar", 2);
> COMMIT;
> -- 399.1666… → 399.17 (correct SAR-halala rounding of a denormalized MRR metric).
> ```
>
> (Note: the `Deal` model `@@map`s to table `"CustomerPropertyInterest"` — its `value`
> column was 0-at-risk. The precheck SQL below already uses the mapped name.)

For each money column below, this finds any value that would lose data — i.e. more than
12 digits left of the point, **or** more than 2 digits right of it. Any non-zero count is
a blocker: reconcile that row before pushing.

```sql
-- Helper predicate per column (substitute "Table"."col"):
--   abs(trunc(col)) >= 10^12      → > 12 integer digits  (overflow on numeric(14,2))
--   col <> round(col, 2)          → > 2 decimal places   (silent truncation)

SELECT 'Organization.capitalAmountSar' AS col, count(*) FROM "Organization"
  WHERE "capitalAmountSar" IS NOT NULL AND (abs(trunc("capitalAmountSar")) >= 1e12 OR "capitalAmountSar" <> round("capitalAmountSar",2))
UNION ALL SELECT 'Unit.price',        count(*) FROM "Unit"        WHERE "price"        IS NOT NULL AND (abs(trunc("price"))        >= 1e12 OR "price"        <> round("price",2))
UNION ALL SELECT 'Unit.markupPrice',  count(*) FROM "Unit"        WHERE "markupPrice"  IS NOT NULL AND (abs(trunc("markupPrice"))  >= 1e12 OR "markupPrice"  <> round("markupPrice",2))
UNION ALL SELECT 'Unit.rentalPrice',  count(*) FROM "Unit"        WHERE "rentalPrice"  IS NOT NULL AND (abs(trunc("rentalPrice"))  >= 1e12 OR "rentalPrice"  <> round("rentalPrice",2))
UNION ALL SELECT 'Customer.budget',   count(*) FROM "Customer"    WHERE "budget"       IS NOT NULL AND (abs(trunc("budget"))       >= 1e12 OR "budget"       <> round("budget",2))
UNION ALL SELECT 'Reservation.amount',        count(*) FROM "Reservation" WHERE "amount"        IS NOT NULL AND (abs(trunc("amount"))        >= 1e12 OR "amount"        <> round("amount",2))
UNION ALL SELECT 'Reservation.depositAmount', count(*) FROM "Reservation" WHERE "depositAmount" IS NOT NULL AND (abs(trunc("depositAmount")) >= 1e12 OR "depositAmount" <> round("depositAmount",2))
UNION ALL SELECT 'Contract.amount',           count(*) FROM "Contract" WHERE "amount"          IS NOT NULL AND (abs(trunc("amount"))          >= 1e12 OR "amount"          <> round("amount",2))
UNION ALL SELECT 'Contract.securityDeposit',  count(*) FROM "Contract" WHERE "securityDeposit" IS NOT NULL AND (abs(trunc("securityDeposit")) >= 1e12 OR "securityDeposit" <> round("securityDeposit",2))
UNION ALL SELECT 'Contract.grossAmount',      count(*) FROM "Contract" WHERE "grossAmount"     IS NOT NULL AND (abs(trunc("grossAmount"))     >= 1e12 OR "grossAmount"     <> round("grossAmount",2))
UNION ALL SELECT 'Contract.discountAmount',   count(*) FROM "Contract" WHERE "discountAmount"  IS NOT NULL AND (abs(trunc("discountAmount"))  >= 1e12 OR "discountAmount"  <> round("discountAmount",2))
UNION ALL SELECT 'Contract.netAmount',        count(*) FROM "Contract" WHERE "netAmount"       IS NOT NULL AND (abs(trunc("netAmount"))       >= 1e12 OR "netAmount"       <> round("netAmount",2))
UNION ALL SELECT 'PaymentPlan.totalAmount',   count(*) FROM "PaymentPlan" WHERE "totalAmount" IS NOT NULL AND (abs(trunc("totalAmount")) >= 1e12 OR "totalAmount" <> round("totalAmount",2))
UNION ALL SELECT 'PaymentPlan.downPayment',   count(*) FROM "PaymentPlan" WHERE "downPayment" IS NOT NULL AND (abs(trunc("downPayment")) >= 1e12 OR "downPayment" <> round("downPayment",2))
UNION ALL SELECT 'PaymentPlanInstallment.amount',     count(*) FROM "PaymentPlanInstallment" WHERE "amount"     IS NOT NULL AND (abs(trunc("amount"))     >= 1e12 OR "amount"     <> round("amount",2))
UNION ALL SELECT 'PaymentPlanInstallment.paidAmount', count(*) FROM "PaymentPlanInstallment" WHERE "paidAmount" IS NOT NULL AND (abs(trunc("paidAmount")) >= 1e12 OR "paidAmount" <> round("paidAmount",2))
UNION ALL SELECT 'Lease.totalAmount',         count(*) FROM "Lease" WHERE "totalAmount" IS NOT NULL AND (abs(trunc("totalAmount")) >= 1e12 OR "totalAmount" <> round("totalAmount",2))
UNION ALL SELECT 'RentInstallment.amount',     count(*) FROM "RentInstallment" WHERE "amount"     IS NOT NULL AND (abs(trunc("amount"))     >= 1e12 OR "amount"     <> round("amount",2))
UNION ALL SELECT 'RentInstallment.paidAmount', count(*) FROM "RentInstallment" WHERE "paidAmount" IS NOT NULL AND (abs(trunc("paidAmount")) >= 1e12 OR "paidAmount" <> round("paidAmount",2))
UNION ALL SELECT 'MaintenanceRequest.estimatedCost', count(*) FROM "MaintenanceRequest" WHERE "estimatedCost" IS NOT NULL AND (abs(trunc("estimatedCost")) >= 1e12 OR "estimatedCost" <> round("estimatedCost",2))
UNION ALL SELECT 'MaintenanceRequest.actualCost',    count(*) FROM "MaintenanceRequest" WHERE "actualCost"    IS NOT NULL AND (abs(trunc("actualCost"))    >= 1e12 OR "actualCost"    <> round("actualCost",2))
UNION ALL SELECT 'PreventiveMaintenancePlan.estimatedCost', count(*) FROM "PreventiveMaintenancePlan" WHERE "estimatedCost" IS NOT NULL AND (abs(trunc("estimatedCost")) >= 1e12 OR "estimatedCost" <> round("estimatedCost",2))
UNION ALL SELECT 'Plan.priceMonthly', count(*) FROM "Plan" WHERE (abs(trunc("priceMonthly")) >= 1e12 OR "priceMonthly" <> round("priceMonthly",2))
UNION ALL SELECT 'Plan.priceAnnual',  count(*) FROM "Plan" WHERE (abs(trunc("priceAnnual"))  >= 1e12 OR "priceAnnual"  <> round("priceAnnual",2))
UNION ALL SELECT 'Subscription.priceAtRenewal', count(*) FROM "Subscription" WHERE "priceAtRenewal" IS NOT NULL AND (abs(trunc("priceAtRenewal")) >= 1e12 OR "priceAtRenewal" <> round("priceAtRenewal",2))

UNION ALL SELECT 'InvoiceLineItem.vatAmount', count(*) FROM "InvoiceLineItem" WHERE (abs(trunc("vatAmount")) >= 1e12 OR "vatAmount" <> round("vatAmount",2))
UNION ALL SELECT 'InvoiceLineItem.total',     count(*) FROM "InvoiceLineItem" WHERE (abs(trunc("total"))     >= 1e12 OR "total"     <> round("total",2))
```

**Expected:** every `count` is `0`. If any row is non-zero, fix that data first (a value
> 12 integer digits is almost certainly a data-entry bug; > 2 decimals is harmless
rounding that the cast would silently apply — confirm it's acceptable, then proceed).

**VAT-rate columns** recast to `numeric(6,4)` — `0.1500` fits exactly; verify nothing odd:

```sql
SELECT 'Invoice.vatRate', count(*) FROM "Invoice" WHERE "vatRate" <> round("vatRate",4) OR abs(trunc("vatRate")) >= 1e2
UNION ALL
SELECT 'InvoiceLineItem.vatRate', count(*) FROM "InvoiceLineItem" WHERE "vatRate" <> round("vatRate",4) OR abs(trunc("vatRate")) >= 1e2;
-- Expected: 0, 0.  (numeric(6,4) holds 2 integer + 4 fractional digits; a rate is 0.xxxx.)
```

## Step 2 — Coupon-redemption duplicate precheck (run BEFORE `db push`)

QA-SEC-02 adds `@@unique([couponId, organizationId])` to `CouponRedemption`. `db push`
will FAIL to create the unique index if any duplicate `(couponId, organizationId)` pair
already exists. Confirm there are none:

```sql
SELECT "couponId", "organizationId", count(*) AS n
FROM "CouponRedemption"
GROUP BY "couponId", "organizationId"
HAVING count(*) > 1
ORDER BY n DESC;
```

**Expected:** zero rows. If any group has `n > 1`, dedupe first — keep the earliest
redemption, delete the rest (an org redeeming the same coupon twice was the bug this
constraint prevents):

## Step 3 — `prisma db push` (the schema change)

After steps 1–2 are clean, apply the schema. This single push performs:
the money/VAT recasts, the 3 additive nullable `organizationId` columns, the new
composite indexes, the `CouponRedemption` unique constraint, and the 3 cascade→restrict
FK rule changes.

```bash
cd packages/db
# Dry-run first against a prod-like DB — plain push self-aborts and names every blocker:
npx prisma db push
```

- The recasts are `ALTER COLUMN TYPE` — non-destructive given step 1 passed.
- The new columns are **nullable** → no populated-table abort (AGENTS.md §4 hazard avoided).
- Do **not** use `--accept-data-loss` unless the printed diff is provably non-destructive
  (new nullable columns + additive indexes + the unique index validated in step 2).
- `prisma db push` cannot express the org-id backfill or the future NOT-NULL tightening —
  those are steps 4 and 5 below.

## Step 5 — NOT-NULL tightening (DEFERRED — a future release, not v4.28.0)

will tighten these columns to `NOT NULL` (and optionally add the real FK + cascade rule).
**Do not do this in v4.28.0** — it must follow a verified, complete backfill on prod, or
`db push` aborts. When that release lands, its schema flips `organizationId String?` →
`organizationId String` (+ FK) and CI re-pushes; the manual prod step will be:
